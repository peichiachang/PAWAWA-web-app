# PAWAWA 容器設定通用規則

水碗與食碗使用相同的設定框架，規則一致。

---

## 容器設定框架

| 項目 | 水碗 | 食碗 |
|------|------|------|
| 一般容器模式 | 水碗模式 | 食碗模式 |
| 特殊容器模式 | 飲水機模式 | 自動餵食器模式 |
| 體積來源（一般） | 測量尺寸 / 側面輪廓推估 | 測量尺寸 / 側面輪廓推估 |
| 體積來源（特殊） | 使用者輸入規格容量（ml） | 使用者輸入每份克數（g） |
| 基準設定 | 滿水基準（一次性） | 滿碗基準（一次性） |
| 記錄方式 | W0 / W1 拍照標記三條線 | T0 / T1 拍照標記三條線 |
| 計算結果 | 一般模式標示「僅供參考」 | 一般模式標示「僅供參考」 |
| 上限保護 | 結果不可超過 volumeMl | 結果不可超過容器容量 |

---

## 模式說明

### 一般容器模式（水碗模式 / 食碗模式）

適用於一般碗、杯等容器。

**輸入方式（二選一）：**
- 測量尺寸：輸入高度與直徑，系統用公式推估容積
- 側面輪廓：拍攝側面照片，AI 識別輪廓後積分推估容積

**注意：** 推估值有幾何誤差（約 ±5~15%），結果標示「僅供參考」

---

### 特殊容器模式（飲水機模式 / 自動餵食器模式）

適用於有明確規格容量的容器。

**飲水機模式：**
- 使用者輸入規格容量（ml）
- 不知道容量？查看產品底部或包裝盒規格標示

**自動餵食器模式：**
- 使用者輸入每份克數（g）與每日次數
- 不知道每份克數？查看自動餵食器設定或產品說明書

## 滿量基準設定（兩種模式共用，一次性）

### 顯示狀態

**已設定：**
```
滿量基準
┌─────────────────────────────────────────┐
│ ✅ 已設定（2025/03/05）     [重新設定]  │
└─────────────────────────────────────────┘
```

**未設定：**
```
滿量基準
┌─────────────────────────────────────────┐
│ ⚠️ 尚未設定                [開始設定]  │
│ 設定後才能開始記錄                      │
└─────────────────────────────────────────┘
```

### Step 1：說明頁

```
設定滿量基準

只需要做一次。

① 請先將容器裝到滿的狀態
② 拍照標記三條線
③ 系統將以此作為容量上限的對應基準

─────────────────────────────
⚠️ 請確認是裝到滿再拍照

─────────────────────────────

[開始拍照]        [取消]
```

### Step 2：拍照與標記

```
設定滿量基準

請拖曳三條線分別對齊：
• 頂線：觀察窗或碗口的上緣
• 底線：觀察窗或碗的底部
• 水位／食物線：目前的位置（滿的狀態）

[重拍]        [確認標記]
```

### Step 3：完成頁

```
✅ 滿量基準設定完成

設定時間：2025/03/05 12:30

之後每次記錄都會以此作為計算基準。

[完成]
```

## 計算邏輯（通用）

```javascript
// 以滿量基準換算百分比
levelPct = (currentY - fullY) / (bottomY - fullY)
amountValue = (1 - levelPct) × maxVolume

// 上限保護
amountValue = Math.min(amountValue, maxVolume)

// 實際攝取
actualIntake = max(0, T0_Value - T1_Value)
```

**maxVolume 來源：**
- 一般模式（水碗 / 食碗）：系統推估值，標示「僅供參考」
- 特殊模式（飲水機 / 自動餵食器）：使用者輸入的規格值

## 邊緣案例（通用）

| 情況 | 處理方式 |
|------|---------|
| 尚未設定基準就開始記錄 | 提示「請先完成滿量基準設定」，跳轉設定流程 |
| 修改容量後未重新設定 | 黃色提示「容量已變更，建議重新設定基準」，不強制 |
| 計算結果超過最大容量 | 顯示「⚠️ 計算結果超過容器容量，請重新標記」，強制重拍 |
| T1 / W1 高於 T0 / W0 | 實際攝取 = 0，不顯示負值 |
| T0 / W0 高於滿量基準 | 顯示「⚠️ 計算結果超過容器容量，請重新標記」，強制重拍 |
