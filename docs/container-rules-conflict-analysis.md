# 容器設定通用規則 — 衝突與對齊分析

對照 `docs/container-setup-general-rules.md` 與目前程式實作，整理衝突與建議對齊方式。

---

## 一、與現狀一致的部分（無衝突）

| 規則 | 現況 | 說明 |
|------|------|------|
| 水碗：水碗模式 / 飲水機模式 | ✅ 已實作 | VesselCalibrationModal 用途=水碗時可選 水碗模式／飲水機模式 |
| 飲水機：使用者輸入容量 (ml) | ✅ 已實作 | 飲水機模式僅輸入 飲水機容量 (ml) + 滿量基準 |
| 水碗：滿水基準一次性、Step 1～3 | ✅ 已實作 | FullWaterCalibrationModal，三步驟文案可再對齊「滿量」用語 |
| W0/W1 拍照標記三條線 | ✅ 已實作 | WaterLevelMarker，滿水校準與 W0/W1 皆用三條線 |
| 上限保護、T1>T0=0、超過容量強制重拍 | ✅ 已實作 | useHydration 已做 volumeMl cap 與錯誤提示 |
| 容量變更黃色提示 | ✅ 已實作 | 「容量已變更，建議重新設定水位基準」 |
| 尚未設定基準不給記錄 | ✅ 已實作 | useHydration 檢查 fullWaterCalibration / vesselType |

## 二、用語差異（可統一，非邏輯衝突）

| 規則用語 | 目前程式 | 建議 |
|----------|----------|------|
| 滿量基準（通用） | 水碗用「水位基準」 | 可選：UI 改為「滿量基準」或保留「水位基準」並在說明中對應「滿水基準」 |
| 滿碗基準（食碗） | 食碗目前無基準設定 | 待實作食碗滿碗基準時，標題可用「滿量基準」或「滿碗基準」 |
| Step 1「裝到滿」 | 目前「滿水狀態」 | 水碗可保留「滿水」；食碗未來用「裝到滿」 |

結論：**無強制衝突**，僅建議之後新畫面對齊「滿量基準」或「滿碗／滿水」子標題。

## 三、食碗與現況的差異（需補齊或取捨）

### 3.1 食碗缺少「容器類型」二選一

- **規則**：食碗 = 食碗模式（一般碗）| 自動餵食器模式（輸入每份克數 + 每日次數）。
- **現況**：VesselCalibrationModal 在 用途=食碗 時只有「輸入方式」測量尺寸／側面輪廓／已知容量，**沒有**「食碗模式 vs 自動餵食器模式」的選項。
- **衝突**：規則要求食碗也像水碗一樣先選「一般 vs 特殊」。
- **建議**：
  - 在 用途=食碗 時，比照水碗增加 **容器類型**：**食碗模式** | **自動餵食器模式**。
  - **食碗模式**：維持現有 測量尺寸／側面輪廓（可保留或拿掉「已知容量」，改由自動餵食器模式專用）。
  - **自動餵食器模式**：僅輸入 **每份克數 (g)**、**每日次數**，再加 **滿碗基準**（一次性），不填幾何尺寸。

### 3.2 食碗缺少「滿碗基準」（一次性）

- **規則**：食碗也有滿碗基準，與水碗滿水基準同為「滿量基準」、一次性設定。
- **現況**：只有水碗有 FullWaterCalibrationModal；食碗**沒有**滿碗校準或三條線基準。
- **衝突**：若完全對齊規則，食碗也需滿碗基準與後續 T0/T1 用同一套計算。
- **建議**：
  - 新增「滿碗基準」流程（可共用 FullWaterCalibrationModal 元件，傳入 type: 'water' | 'food'，僅文案與儲存 key 不同）。
  - 儲存結構可沿用 `fullWaterCalibration` 形狀，或另加 `fullBowlCalibration`（若希望水/食分開 key）。
  - 先對齊 **自動餵食器模式**：必設滿碗基準後才能用該食碗記錄；**食碗模式**可列為第二階段（一般碗是否強制滿碗基準可再訂）。

### 3.3 食碗 T0/T1 記錄方式：三條線 vs 目前 AI 比對

- **規則**：記錄方式為「T0 / T1 拍照標記三條線」，計算用 levelPct 與滿量基準。
- **現況**：食物記錄是 T0/T1 **拍照**，由 **AI 比對前後影像** 估克數，**沒有**三條線標記與滿碗基準換算。
- **衝突**：若完全對齊規則，食物記錄應改為「T0/T1 各拍一張 → 各標三條線 → 用滿碗基準換算克數」。
- **建議**（二選一或分階段）：
  - **方案 A（完全對齊）**：食碗也引入「T0/T1 各標三條線」+ 滿碗基準換算（類似 W0/W1），AI 比對改為選配或僅用於輔助。
  - **方案 B（並存）**：保留現有 AI 比對流程為「食碗模式」的其中一種記錄方式；另在「自動餵食器模式」下強制使用「T0/T1 三條線 + 滿碗基準」。
  - 若採方案 B，需在規則或產品說明中註明：何種容器模式用三條線、何種用 AI 比對，避免與文件矛盾。

### 3.4 自動餵食器：每份克數與「每日次數」

- **規則**：自動餵食器模式輸入「每份克數（g）與每日次數」。
- **現況**：VesselCalibration 有 `defaultPortionGrams`（每份克數），**沒有**「每日次數」欄位。
- **衝突**：規則有「每日次數」，程式未存。
- **建議**：
  - 在 VesselCalibration 新增選用欄位，例如 `dailyPortionCount?: number`（每日幾份）。
  - 自動餵食器模式 UI：必填每份克數，選填每日次數；若未來有「今日總克數」預設值，可依 每份×每日次數 帶入。

### 3.5 一般模式結果標示「僅供參考」

- **規則**：一般容器模式（測量尺寸／側面輪廓）之計算結果標示「僅供參考」。
- **現況**：飲水與食物結果區塊未明確標「僅供參考」。
- **衝突**：屬 UI 文案，易補齊。
- **建議**：當 vessel 為 測量尺寸 或 側面輪廓（非 飲水機／非 自動餵食器）時，在結果區塊加一小行「僅供參考」（水碗／食碗皆適用）。

---

## 四、實作優先順序建議

1. **無衝突、可做文案微調**  
   - 水碗 Step 1～3 的「滿水」可改為「裝到滿」或加註「滿量」；  
   - 必要時將「水位基準」改為「滿量基準」或並列說明。

2. **食碗對齊通用框架（建議優先）**  
   - 用途=食碗 時增加 **容器類型**：食碗模式 | 自動餵食器模式。  
   - **自動餵食器模式**：輸入每份克數 (g)、每日次數（選填）、滿碗基準（一次性）；  
   - 儲存：`defaultPortionGrams`、`dailyPortionCount?`、滿碗基準結構（可共用或獨立 key）。

3. **滿碗基準與 T0/T1 計算（依產品取捨）**  
   - 若採用「T0/T1 三條線 + 滿碗基準」：  
     - 滿碗基準 UI（可共用 FullWaterCalibrationModal）。  
     - T0/T1 拍照後各進入「標記三條線」流程，計算公式與規則一致。  
   - 若保留現有 AI 比對：  
     - 在規則或說明中註明「食碗模式」可用 AI 估克數，「自動餵食器模式」用滿碗基準 + 三條線，避免與通用規則字面衝突。

4. **「僅供參考」標示**  
   - 一般模式（水/食）結果區塊加上「僅供參考」文案。

---

## 五、總結

- **水碗**：與通用規則已大致一致，僅用語可微調為「滿量基準」。
- **食碗**：  
  - 需補「容器類型」與「自動餵食器模式」；  
  - 需補「滿碗基準」與（若對齊規則）「T0/T1 三條線」流程；  
  - 需補「每日次數」欄位與「僅供參考」標示。  

依上述順序對齊後，即可與「PAWAWA 容器設定通用規則」一致且無衝突。
